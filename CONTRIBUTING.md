# Contributing to gitlore

Thanks for your interest in contributing! The easiest and most impactful way to contribute is by **adding new narrative styles**.

## Adding a Style

Each style is a single object in `src/generate/styles.ts`. No other code changes needed.

### 1. Pick a style

Think about narrative voices that would make git history fun to read:
- Literary genres: noir, fairy tale, gothic horror, sci-fi captain's log
- Commentary styles: nature documentary, sports commentary, war correspondent
- Formats: podcast transcript, letter to a friend, conference talk

### 2. Add it to `styles.ts`

```typescript
"your-style": {
  name: "Your Style Name",
  description: "One-line description for --help output.",
  strapline: "Generated by gitlore — your catchy strapline here.",
  systemPrompt: `Your system prompt here. This is the most important part.

Guidelines for writing good style prompts:
- Be specific about voice, structure, and tone
- Tell the LLM what narrative structure to follow
- Reference actual git data (commits, diffs, timestamps, authors)
- Include what to emphasize and what to skip
- Give concrete examples of the kind of writing you want`,
},
```

### 3. Update the CLI help

Add your style name to the `--style` option list in `src/cli/index.ts`.

### 4. Test it

```bash
cd any-git-repo
npx tsx /path/to/gitlore/bin/gitlore.ts --since "1 month ago" --style your-style
```

The output is a prompt — paste it into any LLM to see the narrative.

### 5. Submit a PR

Include a sample output in your PR description so reviewers can see the style in action.

## Other Contributions

- **Bug fixes**: PRs welcome. Include a description of the bug and how to reproduce.
- **Enrichment providers**: Want to add Jira, Linear, or other ticket integrations? Open an issue first to discuss the approach.
- **Core improvements**: For significant changes, open an issue to discuss before writing code.

## Development

```bash
npm install
npm run lint          # Type-check
npm run dev -- --since "1 week ago" --style comedy   # Run locally
npm test              # Run tests
```

## Code Style

- TypeScript strict mode
- No unnecessary abstractions
- Keep it simple — this is a CLI tool, not a framework
